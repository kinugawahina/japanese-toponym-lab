# japanese-toponym-lab

日本の地名に含まれる漢字の分布から、地域差や文化圏を探索する個人研究プロジェクト。

## Current status

- Geolonia Japanese Addresses を利用
- 都道府県別の地名要素頻度行列を作成
- PCA による初期可視化を実装

python src/build_feature_matrix.py
python src/run_pca.py

## 特徴量設計メモ

初期のPCAでは、地名文化圏というよりも、行政制度や住所表記に由来する特徴が強く出た。

特に市区町村単位の分析では、以下のような漢字が主成分を大きく支配した。

- 市
- 区
- 町
- 村
- 郡
- 大
- 字
- 丁
- 目

これらは地名の歴史的・文化的特徴というより、近代的な行政区分や住所管理体系を反映している可能性が高い。

そのため、`municipality_v3` 以降では、これらの行政・住所管理由来の漢字を特徴量から除外する。

一方で、`小` は除外しない。`小字` の一部として現れる可能性はあるが、`小岩`、`小松`、`小倉`、`小山` など、地名語彙として意味を持つ例も多いためである。

このプロジェクトでは、単に分類精度を上げることではなく、地名文化圏の解釈に資する特徴量を作ることを重視する。

## Feature Engineering History

### v1

- 全漢字を利用
- 都道府県単位

結果:
- レア漢字が主成分を支配

### v2

除外:
- 一〜十
- 丁
- 目
- 番
- 号

結果:
- 北海道・東北軸が出現

### v3

追加除外:
- 市
- 区
- 町
- 村
- 郡
- 大
- 字

結果:
- 行政制度由来の軸を低減

### v4

追加除外:
- 第
- 割
- 地
- 条
- 線

結果:
- 岩手の地割表記と北海道の区画表記の影響を低減
- 北海道固有地名要素が明瞭化

### 地名要素版PCA

漢字単体ではなく、`沢`・`谷`・`原`・`浜`・`浦`・`崎`・`別`・`幌` などの地名要素を特徴量として、市区町村単位のPCAを行った。

初期結果では、`別`・`幌` などの要素により北海道の自治体が強く分離した。また、`浦`・`浜`・`崎` などは沿岸的地名要素としてまとまり、`沢`・`峠`・`滝` などは内陸・山地的地名要素として作用している可能性がある。

北海道の信号が非常に強いため、次の実験では北海道を除外し、本州・四国・九州・沖縄内部の地域構造を探索する。
## 分析メモ

### 漢字頻度分析

初期段階では、地名を構成する漢字を都道府県別・市区町村別に集計し、PCAによる可視化を行った。

その結果、北海道に特徴的な `幌`・`別`・`札` などの漢字が強く現れ、地名漢字だけでも地域差を抽出できる可能性が確認された。

一方で、初期の特徴量では以下のようなノイズも確認された。

- レア漢字による過剰な分離
- `一`・`二`・`三`・`丁`・`目` などの丁目・番地由来の特徴
- `市`・`区`・`町`・`村`・`郡` などの行政区分由来の特徴
- `第`・`割`・`地`・`条`・`線` などの住所制度由来の特徴

これらは地名文化圏そのものというより、近代的な行政制度や住所管理体系を反映しているため、後続バージョンでは除外した。

### 地名要素分析

漢字単体ではなく、`沢`・`谷`・`原`・`浜`・`浦`・`崎`・`別`・`幌` などの地名要素を特徴量として、市区町村単位のPCAを行った。

北海道を含む分析では、`別`・`幌` などにより北海道の自治体が強く分離した。

北海道を除外した分析では、以下のような軸が見え始めた。

- `沢`・`館`・`舘`・`峠`・`滝` などによる東北・山地的な軸
- `浦`・`浜`・`崎`・`瀬` などによる沿岸的な軸

この結果は、地名要素だけから日本列島内部の地域構造をある程度再構成できる可能性を示している。

### 次の課題

次の段階では、PCAによる軸の解釈だけでなく、クラスタリングを用いて地名要素に基づく地域区分を抽出する。

特に以下を検証する。

- 北海道・東北・沿岸部・山地部などが自然にクラスタとして分離するか
- 地名要素だけから、既存の行政区画とは異なる文化圏が見えるか
- 将来的にはアイヌ語地名、琉球語地名、韓国・中国の地名要素との比較へ拡張できるか
